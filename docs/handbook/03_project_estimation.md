# プロジェクト規模・工数見積もり (PlanMaker)

現在の「学習計画ツール (PlanMaker)」を、プロのエンジニアが業務として請け負って開発した場合の規模感と見積もり目安です。
※あくまで一般的なWeb開発相場（日本国内、準委任または請負）をベースにした概算です。

## 📊 プロジェクト規模の概要

*   **規模感**: 小規模〜中規模 Webアプリケーション
*   **開発期間**: 約 1.5ヶ月 〜 2.5ヶ月
*   **想定工数**: 2.0 〜 3.0 人月 (1人がフル稼働で2〜3ヶ月)
*   **想定費用**: 160万円 〜 300万円 (単価 80〜100万円/月 と仮定)

## 💡 このアプリの「開発のツボ」

このアプリが見た目以上に「しっかり作る必要がある」ポイント（＝工数がかかる部分）は以下の3点です。

1.  **カリキュラムエディタ (実装済)**
    *   ドラッグ＆ドロップでグリグリ動かす機能は、普通の入力フォームより作るのが大変です。（完了しているので大きな山は越えました）
2.  **データ連携 (これから)**
    *   「計画した単元」を「日々の授業」で呼び出し、「終わったらステータスを変えて」「進捗グラフに反映する」という、データ同士の連動ロジックが複雑になりがちです。
3.  **ローカルファースト (Dexie.js)**
    *   サーバーを使わずにブラウザ内にデータを保存する特殊な作りをしているため、データの整合性を守る部分に気を使います。

## 💰 費用を抑える・期間を短くするには？

もし「もっと安く、早く」を目指すなら、以下の工夫が有効です。

*   **デザインをテンプレート化**: 既存のコンポーネント集 (shadcn/uiなど) をそのまま使い、独自のデザインこだわりを捨てる。（現在はこれを採用しています）
*   **機能を削る**: 「AI自動生成」や「複雑なグラフ」を後回しにし、まずは「記録して一覧で見るだけ」に絞る。
