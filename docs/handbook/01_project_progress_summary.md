# 学習計画ツール開発進捗レポート (2026-02-14 現在)

エンジニアではない方向けに、現在までの「学習計画ツール」の開発進捗をまとめました。
専門用語をなるべく避けて、「何ができるようになったか」「どんな価値があるか」を中心に説明します。

## 🚀 達成したこと：3つの大きな成果

これまでの開発で、**アプリの土台作り**と、**最も重要な2つの機能（生徒管理・カリキュラム作成）**の実装が完了しました。

### 1. アプリの土台とデータ保存の仕組みが完成
まず、アプリがウェブブラウザ上で動くための基盤を作りました。
さらに、**「データが消えない」仕組み（データベース）**をブラウザの中に構築しました。
*   **ポイント**: インターネットに繋がっていなくても動く「ローカルファースト」な設計になっているため、データのプライバシーが守られ、動作も高速です。
*   **テスト機能**: ダッシュボードにある「Seed」ボタンを押すと、実験用のデータ（生徒：Taro Yamadaさんなど）が一瞬で作られます。これにより、データの動作確認が簡単に行えます。

### 2. 生徒管理機能：生徒を登録し、リストで見れる
生徒一人ひとりの情報を管理する機能ができました。
*   **生徒一覧**: 登録されている生徒全員がリストで表示されます。
*   **新規登録**: 新しい生徒の名前、学年、目標などを入力して登録できます。
    *   *確認済み*: 実際に「Hanako Tanaka」さんを登録し、リストに追加されることを確認しました。

### 3. カリキュラム作成機能：ドラッグ＆ドロップで計画作り
生徒ごとの学習計画（カリキュラム）を作るためのエディタ画面を作りました。これが今回の目玉機能です。
*   **ドラッグ＆ドロップ**: マウスで単元（レッスン）の箱をつかんで、順番を自由に入れ替えられます。直感的に操作できます。
*   **単元の追加**: 「二次関数」「5コマ」のように、教科と単元名を入力して追加できます。
*   **AI自動生成（体験版）**: 「AI自動生成」ボタンを押すと、サンプルの単元リストが自動で提案・追加されます。AIがプランを作ってくれる未来の機能をイメージして作りました。
