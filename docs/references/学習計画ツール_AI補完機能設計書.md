1. AI補完機能の概要

本ツールでは、講師の入力負担を軽減しつつ精度の高い学習計画を作成するため、AI補完機能を導入する。補完には2つのアプローチがあり、状況に応じて使い分ける。

📊 過去データベース

🔄 ハイブリッド

7.2 コンテキストの自動生成

授業記録やテスト結果から、AIが自動的にコンテキストを生成・更新する。

自動生成される分析例

【学習特性】
・計算系単元の習得が早い（平均2.1回で完了、全体平均3.0回）
・図形系単元は苦手傾向（平均4.2回、理解度平均3.1）
・文章題で躓きやすい

【進捗パターン】
・週1回ペースで安定して進められる
・2週間空くと前回内容の復習が必要

【テスト傾向】
・未学習範囲でも基礎問題は30%程度取れる
・応用問題は学習済み単元でも落としやすい

7.3 講師によるコンテキスト補正

AIの自動分析を講師が確認・補正できる機能を提供する。

AIでは把握できない情報を明示的に追加

特定の要素を重視/軽視するよう調整

講師補正の例

【講師による補正】
・「図形が苦手」とAIは分析しているが、最近は克服傾向にある → 重み付けを下げる
・部活の大会シーズンは集中力が下がりやすい → 特記事項として追加
・この生徒は褒めると伸びるタイプ → コメント生成時の参考情報として追加
