作成日: 2026/2/13

1. 本書の目的

本書は、学習計画管理ツールのAI駆動開発を成功させるために必要なドキュメントを整理し、作成計画を定める。

特に以下の点に注力する：

・AIが正確にコードを生成できる構造化された仕様の提供
・セキュリティリスクの事前把握と対策の明文化
・段階的な開発を支援するドキュメントの優先順位付け

2. セキュリティリスク分析

3. セキュリティ対策方針

AI駆動開発において、AIに以下のセキュリティ方針を遵守させる。

3.1 データ保護の原則

原則

具体的な実装指針

最小権限の原則

必要最小限のデータのみ扱う。生徒の氏名・住所等は本ツールでは一切扱わない

データの暗号化

LocalStorageへの保存時はAES-256で暗号化。鍵はユーザーのパスワードから派生

センシティブデータの分離

コンテキスト情報は通常データと分離して保存・管理

明示的な削除

生徒削除時は関連する全データ（履歴、コンテキスト含む）を完全削除

3.2 AI補完のセキュリティ

原則

具体的な実装指針

ローカルファースト

AI処理は可能な限りブラウザ内で実行。外部API利用は最小限に

匿名化必須

外部AIに送信する場合、生徒番号も含め識別子を完全除去

送信内容の明示

外部送信時はユーザーに送信内容を表示し、明示的な同意を取得

ローカルモデル優先

WebLLM等のブラウザ内AI活用を検討（Phase 3以降）

3.3 入力検証とサニタイズ

対象

実装指針

全てのユーザー入力

ホワイトリスト方式で許可文字を限定、HTMLタグは除去

JSONインポート

スキーマ検証を必須とし、不正なデータ構造は拒否

コメント・メモ欄

長さ制限、特殊文字のエスケープ

数値入力

型チェックと範囲チェックを必須

3.4 AIへの指示テンプレート

コード生成時、AIに以下のセキュリティ要件を毎回伝達する：

【セキュリティ必須要件】
1. ユーザー入力は必ずサニタイズしてから使用すること
2. LocalStorageへの保存は暗号化関数を経由すること
3. 外部APIへのデータ送信は禁止（明示的に許可された場合を除く）
4. エラーメッセージにシステム内部情報を含めないこと
5. 依存ライブラリは最小限に、既知の脆弱性がないことを確認
6. dangerouslySetInnerHTML等の危険なAPIは使用禁止
