1. プロジェクト概要

1.1 目的

個別指導塾における生徒の学習計画を効率的に管理し、パーソナライズされた指導を実現するためのツールを開発する。

1.2 対象ユーザー

個別指導塾の講師（複数の生徒を担当）

1.3 基本コンセプト

項目

内容

授業形態

1対1の個別指導

授業頻度

週1回（基本）、季節講習時は増加

計画の特徴

毎回の入力ごとに修正し、現状にフィットさせる継続的な改善

プライバシー

生徒は番号で管理（個人情報は別ファイルで管理）

AI活用

入力補完・予測・提案によりユーザー負担を軽減

3. 進捗ステータス定義

ステータス

説明

次のアクション

未着手

まだ取り組んでいない単元

計画に組み込み、導入を開始

導入済み

説明・解説は完了した状態

演習問題に進む

演習中

問題演習の段階

定着度を確認

定着確認待ち

一通り終わり、テスト等で確認予定

テスト実施後に判定

完了

十分に定着した状態

次の単元へ（定期的に復習チェック）

要復習

時間経過や理解不足で再度取り組み必要

復習を計画に組み込む

保留

他の単元を優先、後回しにしている

タイミングを見て再開

6.3 AI質問機能

補完の精度が不確かな場合や重要な判断が必要な場合にAIがユーザーに確認を求める。

発生条件

質問例

補完の信頼度が低い

「理解度は3で合っていますか？過去データが少ないため確信が持てません」

過去データと矛盾

「前回は理解度5でしたが今回は2に下がっています。特別な事情がありましたか？」

重要な判定

「この単元を『完了』にしますか？復習が必要になる可能性が高いです」

異常値の検出

「予定回数が1回ですが、通常3-4回かかります。本当に大丈夫ですか？」
